# How to Use ReseAIrch

This guide covers the installation and usage of the ReseAIrch Autonomous Agent System. You can run ReseAIrch natively using a Python Virtual Environment, or fully containerized using Docker.

---

## 1. Installation & Setup

### Option A: Docker Installation (Recommended)
Docker handles all system dependencies natively, ensuring the Web Scraper and UI function perfectly.

1. Clone the repository and configure your environment variables:
   ```bash
   git clone https://github.com/WoollyBonsai/ReseAIrch.git
   cd ReseAIrch
   cp .env.example .env
   ```
2. (Optional) Open `.env` and set your specific API keys, or just use the local Ollama instance running on `http://host.docker.internal:11434`.

### Option B: Native Python Installation (Fedora/Linux)
If you prefer running directly on your host machine:

1. Clone the repository and navigate into it:
   ```bash
   git clone https://github.com/WoollyBonsai/ReseAIrch.git
   cd ReseAIrch
   ```
2. Create and activate a virtual environment, then install dependencies:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   playwright install chromium
   ```

---

## 2. Usage Instructions

### The Premium Web Interface
If you want to use the beautifully styled Web UI to manage your model engines and scraping depths dynamically:

* **Using Docker:**
  ```bash
  docker compose --profile web up --build
  ```
* **Using Native Python:**
  ```bash
  export PYTHONPATH=$(pwd)
  python src/ui/web.py
  ```
* **Access the UI:** Open your browser and navigate to `http://localhost:8080`. Type your prompt, select your model, and watch the agents go to work!

### The Command Line Interface (CLI)
If you prefer the raw speed of a terminal environment:

* **Using Docker:**
  ```bash
  docker compose --profile cli run --rm reseairch-cli run "Your prompt here" --depth 2
  ```
* **Using Native Python:**
  ```bash
  export PYTHONPATH=$(pwd)
  source venv/bin/activate
  python src/ui/cli.py run "Your prompt here" --depth 2
  ```

---

## 3. Where is My Data Saved?

When the Harvester and Synthesizer complete a run, check the `workspace/` directory:

1. **`workspace/raw/`**: Contains the raw, unprocessed HTML/Text scrape dumps of every URL visited.
2. **`workspace/processed/`**: Contains the synthesized, categorized output files (e.g., `nmap_scanning_methods.md`, `training_data.jsonl`) generated by the AI based on your formatting demands.
3. **`workspace/chroma/`**: Contains the persistent SQLite Vector Database where all historical scraping memory is kept.

## 4. Hardware Optimization
ReseAIrch was built on an AMD Ryzen 7 AI 250 architecture. It leverages local inference beautifully. The recommended model for complex synthesis is `ollama/mistral-nemo` due to its massive 128k context window, which is ideal for summarizing huge chunks of HTML memory.
